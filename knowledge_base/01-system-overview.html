DaFlavors System Overview

What is DaFlavors?

DaFlavors is a comprehensive Django-based event management platform that connects clients with venue and catering service providers. The system facilitates the entire event planning process from venue booking to catering arrangements.

System Architecture

Core Components

  • accounts/ - User management and authentication system
  • venues/ - Venue listings and management
  • caterers/ - Catering service management with menu systems
  • bookings/ - Event booking and management workflow
  • core/ - Site-wide functionality, admin interfaces, and settings

User Types

1. Clients

End users who book venues and catering services for their events.

2. Providers

Business owners who offer venues and/or catering services. They can manage multiple venues and caterers under one account.

3. Administrators

Staff members who monitor the platform, manage service quality, and handle suspensions.

Key Features

  • Venue Management: Listing, booking, and availability management
  • Catering Services: Menu packages, dietary accommodations, service types
  • Booking System: Complete event booking workflow with quotes and payments
  • Admin Monitoring: Custom admin interface for service regulation
  • Dark Theme UI: Modern, professional interface design
  • Multiple Services: Providers can operate multiple venues/caterers
  • Event Types: Weddings, corporate events, birthdays, etc.
  • Menu Customization: Flexible menu packages with course selections

Database Models Hierarchy

User (CustomUser) ├── ProviderProfile (for providers) ├── Venue(s) (multiple venues per provider) │ ├── VenueImages │ └── VenueAvailability └── Caterer(s) (multiple caterers per provider) ├── CatererImages ├── MenuPackages │ └── PackageItems ├── MenuItems └── CatererAvailability Booking ├── Client (User) ├── Venue (optional) ├── Caterer (optional) ├── MenuSelection ├── CourseSelection ├── Quote ├── Payment └── Messages

URL Structure

Admin URLs

/admin-panel/ - Custom admin dashboard

Technology Stack

  • Backend: Django 4.x with Python
  • Database: PostgreSQL/SQLite
  • Frontend: HTML5, CSS3, JavaScript, Tailwind CSS
  • Authentication: Django's built-in auth system
  • File Storage: Django's file handling for images
  • Admin Interface: Custom admin + Django admin

Business Logic Flow

1. Provider Registration: Business owners register as providers

2. Service Creation: Providers create venue and/or caterer profiles

3. Client Discovery: Clients browse and search for services

4. Booking Process: Clients create bookings for events

5. Quote Generation: Providers respond with quotes

6. Payment Processing: Booking confirmation with payments

7. Admin Monitoring: Continuous quality control and regulation
